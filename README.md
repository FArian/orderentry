# ZetLab OrderEntry — Frontend

Next.js 15.5 web application for laboratory order entry at ZLZ Zentrallabor AG.
Clinicians search patients, browse a FHIR-backed test catalog, and submit diagnostic orders.

**UI language: German (Swiss context)**

---

## Development Setup

### Prerequisites

- Node.js 20.x (see `.nvmrc`)
- Access to a FHIR R4 server (HAPI FHIR) — see [Configuration](#configuration)

### Install & Run

```bash
npm install
npm run dev       # Start dev server with Turbopack at http://localhost:3000
```

### Available Commands

```bash
npm run dev       # Dev server (Turbopack, hot reload)
npm run build     # Production build (outputs standalone for Docker)
npm run start     # Start production server locally
npm run lint      # ESLint (next/core-web-vitals + next/typescript)
npx tsc --noEmit  # Type-check without emitting files
```

## Configuration

Set the FHIR base URL in [`src/config.ts`](src/config.ts):

```ts
export const fhirBase = "https://your-hapi-fhir-server/fhir";
```

Environment variables (create `.env.local` in this directory):

| Variable | Purpose | Default |
|---|---|---|
| `AUTH_SECRET` | HMAC-SHA256 key for session cookies | `dev-secret-change-me` |
| `ALLOW_LOCAL_AUTH` | Allow browser localStorage fallback auth | `false` |
| `NEXT_PUBLIC_FORCE_LOCAL_AUTH` | Force browser-only auth (no server store) | — |
| `NEXT_PUBLIC_APP_VERSION` | Auto-generated by `scripts/write-version.mjs` | — |

## Routes

| Route | Description |
|---|---|
| `/` | Home — links to patient search and orders list |
| `/login` | Login form |
| `/signup` | User registration |
| `/patient` | Patient search with debounced pagination (20/page) |
| `/patient/[id]` | Patient demographics + order history |
| `/order/[id]` | Order entry: tab-based test catalog, specimen selection |
| `/orders` | Global service requests list |

## Key Patterns

- `"use client"` on all interactive pages; layouts and static pages are server components
- `export const dynamic = "force-dynamic"` on root layout
- Dates formatted as `DD.MM.YYYY` (German format, no date library)
- `@/*` path alias maps to `src/*`
- No external state management — `useState`/`useEffect` throughout
- Skeleton rows shown during data fetch (no spinner)

## FHIR Field Mapping — Test Catalog

The order entry form (`/order/[id]`) renders the lab test catalog from FHIR resources.

### Categories (Tabs)

- **Source:** `ActivityDefinition.topic.coding.display`
- **Filter query:** `GET [baseUrl]/ActivityDefinition?topic=MIBI`

### Subgroups & Filters

| UI element | FHIR field |
|---|---|
| Clinical context | `ActivityDefinition.description` |
| Specimen description | `ActivityDefinition.subtitle` |
| LIS code / synonyms | `ActivityDefinition.code.coding.code` |

### Specimen / Material

| UI element | FHIR field |
|---|---|
| Material name | `SpecimenDefinition.typeCollected.text` |
| Container type | `SpecimenDefinition.container.description` |
| Link to specimen | `ActivityDefinition.extension.valueReference` → `SpecimenDefinition` |

### Usability Features (planned / partial)

| Feature | FHIR field |
|---|---|
| Autocomplete search | `ActivityDefinition.code.coding.display` |
| Lab manual link | `ActivityDefinition.relatedArtifact.url` |
| Minimum volume check | `ActivityDefinition.extension.valueQuantity` |
| Reference ranges | `ObservationDefinition` |

## Authentication

Two modes — server-side is the default:

**Server-side (primary)**
- User accounts in `data/users.json` (gitignored)
- Passwords hashed with `crypto.scrypt`
- Sessions signed with HMAC-SHA256 (`AUTH_SECRET`) as httpOnly cookies

**Browser fallback** (`ALLOW_LOCAL_AUTH=true`)
- Users stored in `localStorage`
- Passwords hashed with WebCrypto SHA-256
- Enabled for environments without a writable filesystem
